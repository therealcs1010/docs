AlgoSenpai Adventures - Dev Ops

1. Build Automation

We use Gradle to automate builds of AlgoSenpai Adventures.

2. Continuous Integration

We use Travis CI to perform Continuous Integration on our projects.

3. Coverage Reporting

We use Coveralls to track the code coverage of our projects.

4. Documentation Previews

When a pull request has changes to asciidoc files, you can use Netlify to see a preview of how the HTML version of those asciidoc files will look like when the pull request is merged.

5. Making a Release

Here are the steps to create a new release.

  1. Update the version number in the issue tracker.

  2. Generate a JAR file using Gradle.

  3. Tag the repo with the version number. e.g. v1.0

  4. Create a new release using GitHub and upload the JAR file you created.

6. Managing Dependencies

Gradle downloads the dependencies automatically.
